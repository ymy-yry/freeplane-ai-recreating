# 异步处理架构升级说明文档

## 1. 概述

本文档详细说明了 AI-Build 模块异步处理架构的升级改造，包括架构变更、核心组件、使用方法和最佳实践。

## 2. 升级背景

### 2.1 原架构问题

原架构存在以下问题：

| 问题 | 描述 | 影响 |
|------|------|------|
| **伪异步执行** | BuildTaskScheduler 使用 `executorService.submit()` 同步执行任务 | 无法真正实现异步，UI 仍会阻塞 |
| **重复线程池** | BuildTaskScheduler 和 AsyncBuildTaskExecutor 各自维护线程池 | 资源浪费，架构混乱 |
| **职责不清** | 调度器既负责调度又负责执行 | 违反单一职责原则 |
| **功能闲置** | AsyncBuildTaskExecutor 已实现但未被使用 | 资源浪费 |

### 2.2 升级目标

- 实现真正的异步执行能力
- 统一线程池管理
- 清晰分离调度和执行职责
- 支持任务状态监控和取消

## 4. 核心组件

### 4.1 AsyncBuildTaskExecutor

异步任务执行器，负责实际的任务执行。

**主要方法**：

```java
// 同步执行（向后兼容）
AIServiceResponse executeTask(BuildTask task)

// 异步执行
AsyncTaskResult executeTaskAsync(BuildTask task)

// 带回调异步执行
AsyncTaskResult executeTaskAsync(BuildTask task, TaskExecutionCallback callback)

// 取消任务
boolean cancelTask(String taskId)

// 添加状态监听器
void addTaskStateListener(TaskStateListener listener)
```

**使用示例**：

```java
// 1. 基本异步执行
AsyncBuildTaskExecutor.AsyncTaskResult result =
    AsyncBuildTaskExecutor.getInstance().executeTaskAsync(task);

result.whenComplete((response, error) -> {
    if (error != null) {
        System.out.println("任务失败: " + error.getMessage());
    } else {
        System.out.println("任务成功: " + response.getData());
    }
});

// 2. 带回调
AsyncBuildTaskExecutor.getInstance().executeTaskAsync(task,
    (t, response) -> {
        // 成功处理
        System.out.println("任务完成: " + response.getData());
    },
    (t, error) -> {
        // 错误处理
        System.out.println("任务失败: " + error.getMessage());
    }
);

// 3. 带进度监控
AsyncBuildTaskExecutor.getInstance().executeTaskAsync(task,
    (taskId, oldState, newState) -> {
        System.out.println("状态变更: " + oldState + " -> " + newState);
    }
);
```

### 4.2 AsyncTaskResult

异步任务结果封装类。

**主要方法**：

```java
// 获取任务对象
BuildTask getTask()

// 获取 CompletableFuture
CompletableFuture<AIServiceResponse> getFuture()

// 阻塞获取结果
AIServiceResponse get()

// 带超时获取
AIServiceResponse get(long timeout, TimeUnit unit)

// 添加完成回调
AsyncTaskResult whenComplete(BiConsumer<AIServiceResponse, Throwable> action)

// 添加成功回调
AsyncTaskResult thenAccept(Consumer<AIServiceResponse> action)

// 添加异常处理
AsyncTaskResult exceptional(Consumer<Throwable> action)

// 检查状态
boolean isDone()
boolean isSuccess()
```

**使用示例**：

```java
AsyncTaskResult result = executor.executeTaskAsync(task);

// 链式调用
result.whenComplete((response, error) -> { })
      .thenAccept(response -> { })
      .exceptional(error -> { });

// 超时控制
try {
    AIServiceResponse response = result.get(30, TimeUnit.SECONDS);
} catch (TimeoutException e) {
    System.out.println("任务超时");
}
```

### 4.3 TaskStateListener

任务状态监听器接口。

```java
public interface TaskStateListener {
    void onTaskStateChanged(String taskId, BuildTask task,
                          BuildTask.TaskStatus oldState,
                          BuildTask.TaskStatus newState);
}
```

**任务状态流转**：

```
PENDING -> RUNNING -> COMPLETED
                   -> FAILED
```

### 4.4 BuildTaskExecutor

对 AsyncBuildTaskExecutor 的包装，保持向后兼容。

```java
// 同步执行（向后兼容）
BuildTaskExecutor executor = BuildTaskExecutor.getInstance();
AIServiceResponse response = executor.executeTask(task);

// 异步执行
AsyncBuildTaskExecutor.AsyncTaskResult asyncResult =
    executor.executeTaskAsync(task);
```

## 5. 配置参数

### 5.1 调度参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| temperature | 0.2 | 控制任务选择的随机性 (0.0-2.0) |
| topP | 0.9 | 核采样概率，控制候选集大小 (0.0-1.0) |
| seed | null | 随机种子，null表示随机 |

### 5.2 配置文件

位置：`org/freeplane/plugin/ai/service/scheduling/scheduling.properties`

```properties
# 温度参数 (0.0-2.0)
temperature=0.2

# Top-p 参数 (0.0-1.0)
topP=0.9

# 种子参数 (null表示随机)
seed=null

# Top-k 参数 (null表示无限制)
topK=null

# 存在惩罚 (-2.0 到 2.0)
presencePenalty=0.0

# 频率惩罚 (-2.0 到 2.0)
frequencyPenalty=0.0

# 启用调度控制
enabled=true
```

## 6. 使用场景

### 6.1 场景一：提交任务并异步处理结果

```java
// 提交任务
BuildTask task = BuildTaskScheduler.getInstance().submitTask(
    "generate-mindmap",
    Map.of("topic", "机器学习", "maxDepth", 3)
);

// 异步获取结果
AsyncBuildTaskExecutor.AsyncTaskResult result =
    BuildTaskExecutor.getInstance().executeTaskAsync(task);

result.whenComplete((response, error) -> {
    if (error != null) {
        showError("任务失败: " + error.getMessage());
    } else {
        updateMindMap(response.getData());
    }
});
```

### 6.2 场景二：任务取消

```java
// 提交任务
BuildTask task = scheduler.submitTask("expand-node", request);

// 用户取消
boolean cancelled = scheduler.cancelTask(task.getId());
if (cancelled) {
    showMessage("任务已取消");
}
```

### 6.3 场景三：进度监控

```java
// 添加进度监听器
scheduler.addTaskStateListener((taskId, task, oldState, newState) -> {
    switch (newState) {
        case PENDING:
            showStatus("等待中...");
            break;
        case RUNNING:
            showStatus("执行中: " + taskId);
            break;
        case COMPLETED:
            showStatus("已完成");
            break;
        case FAILED:
            showStatus("失败");
            break;
    }
});
```

### 6.4 场景四：并发任务处理

```java
// 提交多个任务
List<BuildTask> tasks = new ArrayList<>();
for (int i = 0; i < 5; i++) {
    tasks.add(scheduler.submitTask("generate-mindmap",
        Map.of("topic", "主题" + i)));
}

// 并行执行结果处理
List<AsyncBuildTaskExecutor.AsyncTaskResult> results = tasks.stream()
    .map(task -> executor.executeTaskAsync(task))
    .collect(Collectors.toList());

// 等待所有任务完成
CompletableFuture.allOf(
    results.stream()
        .map(r -> r.getFuture())
        .toArray(CompletableFuture[]::new)
).join();
```

## 7. 最佳实践

### 7.1 线程安全

- 使用单例模式获取执行器实例
- 避免在回调中修改共享状态
- 使用 `ConcurrentHashMap` 存储任务状态

### 7.2 资源管理

- 及时调用 `shutdown()` 释放线程池
- 设置合理的队列容量
- 监控线程池状态

```java
// 监控线程池
String poolInfo = executor.getPoolInfo();
System.out.println(poolInfo);
// 输出: Pool: active=2, completed=15, total=20, queue_size=3
```

### 7.3 错误处理

```java
result.whenComplete((response, error) -> {
    if (error != null) {
        Throwable cause = error.getCause();
        if (cause instanceof TimeoutException) {
            // 超时处理
        } else if (cause instanceof NetworkException) {
            // 网络错误处理
        } else {
            // 其他错误处理
        }
    }
});
```

### 7.4 性能优化

- 复用 `AsyncTaskResult` 对象
- 避免频繁创建回调
- 使用批量提交减少调度开销

## 8. 迁移指南

### 8.1 从同步到异步

**之前**：

```java
BuildTask task = scheduler.submitTask("generate-mindmap", request);
AIServiceResponse response = executor.executeTask(task);  // 同步阻塞
```

**现在**：

```java
BuildTask task = scheduler.submitTask("generate-mindmap", request);
AsyncTaskResult result = executor.executeTaskAsync(task);

result.whenComplete((response, error) -> {
    if (error != null) {
        handleError(error);
    } else {
        handleSuccess(response);
    }
});
```

### 8.2 状态监听迁移

**之前**：无状态监听

**现在**：

```java
scheduler.addTaskStateListener((taskId, task, oldState, newState) -> {
    updateUI(taskId, oldState, newState);
});
```

## 10. 常见问题

### Q1: 如何确保任务按优先级执行？

A: 设置 `temperature=0.0` 和 `topP=0.5`，这将严格按优先级排序执行。

### Q2: 如何实现任务超时？

A: 使用 `AsyncTaskResult.get(timeout, unit)` 方法：

```java
try {
    response = result.get(30, TimeUnit.SECONDS);
} catch (TimeoutException e) {
    // 处理超时
}
```

### Q3: 如何实现任务重试？

A: 在回调中检查失败并重新提交：

```java
result.whenComplete((response, error) -> {
    if (error != null && retryCount < MAX_RETRIES) {
        retryCount++;
        executor.executeTaskAsync(task);  // 重试
    }
});
```

### Q4: 如何实现任务链式执行？

A: 使用 `CompletableFuture.thenCompose`：

```java
result.getFuture()
    .thenCompose(response -> {
        BuildTask nextTask = createNextTask(response);
        return executor.executeTaskAsync(nextTask).getFuture();
    })
    .thenAccept(finalResponse -> {
        // 处理最终结果
    });
```
