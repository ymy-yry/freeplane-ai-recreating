## 一、项目概述

### 1.1 项目简介

Freeplane 是一款开源的思维导图编辑器，基于 **Java/Swing + OSGi 插件框架** 构建，支持 Windows、macOS、Linux 三平台。

本项目在原版 Freeplane 1.13.x 基础上进行了以下重大增强：
- ✅ **AI 智能集成**: 基于 LangChain4j 的 AI 对话、工具调用、MCP 协议支持
- ✅ **REST API 桥接层**: 前后端分离架构，支持 Web 端思维导图编辑
- ✅ **Vue3 Web 前端**: 基于 Vue Flow 的现代化思维导图可视化
- ✅ **智能缓冲层**: 自然语言理解、自动模型选择、提示词优化

### 1.2 技术栈

| 层次 | 技术 | 版本 |
|------|------|------|
| **后端语言** | Java 8（核心）/ Java 17（AI 插件） | 8/17 |
| **UI 框架** | Java Swing | - |
| **前端框架** | Vue 3 + TypeScript | 3.5+ |
| **可视化** | Vue Flow | 1.41+ |
| **插件容器** | OSGi (Knopflerfish) | 8.0.11 |
| **构建工具** | Gradle | **9.4.1**（必须 ≥ 9.0，< 10.0） |
| **AI 框架** | LangChain4j | 1.10.0 |
| **脚本引擎** | Apache Groovy | 4.0.27 |
| **Web 服务器** | JDK HttpServer | 内置 |
| **测试框架** | JUnit 4 + AssertJ + Mockito | - |
| **文件格式** | `.mm` (FreeMind 兼容 XML) | - |

## 二、环境准备与构建

### 2.1 安装 JDK

项目需要 **Java 17+**（AI 插件要求 Java 17，其余模块为 Java 8）。

#### 方式 1：使用 SDKMAN（推荐，Linux/macOS）

```bash
# 安装 SDKMAN
curl -s "https://get.sdkman.io" | bash
source "$HOME/.sdkman/bin/sdkman-init.sh"

# 安装 Java 21
sdk install java 21.0.5-zulu
sdk use java 21.0.5-zulu
```

#### 方式 2：Windows 手动安装

1. 下载 [Zulu JDK 21](https://www.azul.com/downloads/) 或 [Oracle JDK](https://www.oracle.com/java/technologies/downloads/)
2. 安装后设置环境变量：
   - `JAVA_HOME` → JDK 安装目录（如 `C:\Program Files\Zulu\zulu-21`）
   - `PATH` → 添加 `%JAVA_HOME%\bin`
3. 验证安装：
   ```powershell
   java -version
   # 应显示: java version "21.x.x"
   ```

### 2.2 安装 Gradle

> ⚠️ **重要**：项目大量使用了 Gradle 9.0 引入的属性和语法，**必须使用 Gradle 9.0 或更高版本**（但 < 10.0）。
> 
> - ✅ **推荐版本**: Gradle 9.4.1
> - ❌ **不支持**: Gradle 8.x 及更早版本
> - ❌ **不兼容**: Gradle 10+（因弃用特性被移除）

#### 方式 1：使用 SDKMAN（Linux/macOS，推荐）

```bash
# 安装 Gradle 9.4.1
sdk install gradle 9.4.1

# 使用指定版本
sdk use gradle 9.4.1

# 设置为默认版本
sdk default gradle 9.4.1
```

#### 方式 2：Windows 手动安装

1. **下载 Gradle 9.4.1**：
   ```
   https://services.gradle.org/distributions/gradle-9.4.1-bin.zip
   ```

2. **解压到本地目录**（如 `C:\gradle\gradle-9.4.1`）

3. **配置环境变量**：
   - 新建系统变量 `GRADLE_HOME` = `C:\gradle\gradle-9.4.1`
   - 编辑 `PATH`，添加 `%GRADLE_HOME%\bin`

4. **验证安装**：
   ```powershell
   gradle -v
   # 应显示: Gradle 9.4.1
   ```

#### 方式 3：Windows 使用包管理器

```powershell
# 使用 Scoop
scoop install gradle@9.4.1

# 或使用 Chocolatey
choco install gradle --version=9.4.1
```

#### 方式 4：手动安装（任意平台）

1. 从 [Gradle 9.4.1 下载页](https://gradle.org/releases/) 下载
2. 解压到本地目录
3. 将 `bin` 目录添加到系统 `PATH`

**验证安装**：
```bash
gradle -v
# 必须显示: Gradle 9.x.x（x ≥ 0）
```

### 2.3 构建命令

```bash
# 编译所有模块
gradle build

# 仅编译核心模块
gradle :freeplane:compileJava

# 运行所有测试
```

## 三、核心架构

### 3.1 启动流程

```
Launcher.create()
  └─ 初始化 Knopflerfish OSGi 框架
       └─ 加载 freeplane bundle（核心）
            └─ 加载所有 freeplane_plugin_* bundles
                 └─ 各插件 Activator.start() 注册扩展点
                      └─ AI 插件启动 REST API + MCP 服务器
```

### 3.2 核心数据模型

#### MapModel（导图模型）

```java
public class MapModel {
    private NodeModel root;              // 根节点
    private Map<String, NodeModel> nodes; // 所有节点索引
    private URL url;                      // 文件位置
    private ExtensionContainer extensions; // 扩展容器
}
```

#### NodeModel（节点模型）

```java
public class NodeModel {
    private String id;                    // 节点唯一标识
    private String text;                  // 节点文本
    private NodeModel parent;             // 父节点
    private List<NodeModel> children;     // 子节点列表
    private boolean folded;               // 是否折叠
    private Side side;                    // 布局位置（左/右）
    private ExtensionContainer extensions; // 扩展（图标、属性、样式等）
}
```

### 3.3 控制器架构

```
Controller (应用控制器)
  └─ ModeController (模式控制器)
       ├─ MMapController (导图编辑控制器) ← 所有写操作必须通过此控制器
       ├─ MTextController (文本编辑控制器)
       ├─ MIconController (图标控制器)
       └─ MNoteController (备注控制器)
```

**关键原则**：所有节点写操作必须通过 `MMapController` 执行，以确保：
- ✅ Swing UI 同步刷新
- ✅ 撤销/重做栈记录
- ✅ 节点监听器触发
- ✅ 数据持久化

### 3.4 扩展点机制

Freeplane 使用 `IExtension` 接口实现可插拔架构：

```java
public interface IExtension {
    // 扩展点生命周期
    void install();
    void uninstall();
}
```

各插件通过 `IModeControllerExtensionProvider` 在 MindMap 模式下注册功能。

## 四、插件系统

### 4.1 插件标准结构

所有插件遵循相同的 OSGi 结构：

```
freeplane_plugin_<name>/
├── src/main/java/
│   └── org/freeplane/plugin/<name>/
│       ├── Activator.java          # OSGi 入口（BundleActivator）
│       └── ...                     # 插件代码
├── src/main/resources/
│   └── preferences.xml             # 偏好设置
└── build.gradle                    # 插件构建配置
```

**build.gradle 关键配置**：
```groovy
ext.bundleActivator = 'org.freeplane.plugin.<name>.Activator'
ext.bundleImports = 'org.freeplane.core.*, org.freeplane.features.*'
ext.bundleExports = 'org.freeplane.plugin.<name>.*'
```

### 4.2 核心插件列表

| 插件 | Bundle 名称 | Java 版本 | 功能描述 |
|------|------------|-----------|---------|
| **AI** | `org.freeplane.plugin.ai` | 17 | AI 对话、工具调用、MCP 服务器、REST API ⭐ |
| **Script** | `org.freeplane.plugin.script` | 8 | Groovy 脚本引擎、节点自动化 |
| **Formula** | `org.freeplane.plugin.formula` | 8 | 电子表格公式计算（类似 Excel） |
| **LaTeX** | `org.freeplane.plugin.latex` | 8 | 数学公式渲染 |
| **Markdown** | `org.freeplane.plugin.markdown` | 8 | Markdown 语法渲染 |
| **CodeExplorer** | `org.freeplane.plugin.codeexplorer` | 8 | Java 代码依赖分析可视化 |
| **BugReport** | `org.freeplane.plugin.bugreport` | 8 | 异常捕获与报告 |
| **OpenMaps** | `org.freeplane.plugin.openmaps` | 8 | 嵌入 OpenStreetMap |
| **SVG** | `org.freeplane.plugin.svg` | 8 | SVG 图形渲染 |

### 4.3 AI 插件特殊架构

AI 插件采用 **双版本兼容设计**：

```
freeplane_plugin_ai/
├── bootstrap/          # Java 8 编译（Java8BootstrapActivator）
│                       # 检测 Java 版本，< 17 显示提示信息
└── main/               # Java 17 编译（PluginRuntimeActivator）
    ├── chat/           # 聊天对话子系统
    ├── tools/          # AI 工具集（增删改查节点）
    ├── mcpserver/      # MCP 协议服务器（Port 6298）
    └── restapi/        # REST API 服务器（Port 6299）⭐
```

**启动逻辑**：
1. `Java8BootstrapActivator` 检查运行时 Java 版本
2. 如果 Java ≥ 17，反射加载 `PluginRuntimeActivator`
3. 如果 Java < 17，显示"AI 功能需要 Java 17+"提示

## 五、AI 功能模块

### 5.1 整体架构

```
用户输入
  ↓
智能缓冲层 (BufferLayer)
  ├─ 需求分析 (MindMapRequirementAnalyzer)
  ├─ 提示词优化 (MindMapPromptOptimizer)
  ├─ 模型选择 (MindMapModelRouter)
  └─ 结果优化 (MindMapResultOptimizer)
  ↓
AI 服务 (AIChatService / LangChain4j)
  ├─ OpenRouter (GPT-4o, Claude 3.5, Llama 3)
  ├─ Google Gemini (gemini-2.0-flash, gemini-3)
  └─ Ollama (本地模型)
  ↓
工具调用 (AIToolSet)
  ├─ 读取节点 (readNodesWithDescendants)
  ├─ 创建节点 (createNodes)
  ├─ 编辑节点 (edit)
  ├─ 删除节点 (deleteNodes)
  ├─ 移动节点 (moveNodes)
  └─ 搜索节点 (searchNodes)
  ↓
思维导图更新 (MMapController)
```

### 5.2 AI 对话系统

#### 核心组件

| 组件 | 文件 | 功能 |
|------|------|------|
| **AIChatPanel** | `chat/AIChatPanel.java` | Swing UI 面板，嵌入右侧标签页 |
| **AIChatService** | `chat/AIChatService.java` | LangChain4j AiServices 封装 |
| **AIChatModelFactory** | `chat/AIChatModelFactory.java` | 根据 provider 创建 ChatModel |
| **AIProviderConfiguration** | `chat/AIProviderConfiguration.java` | 读取 API Key / 地址配置 |
| **AIModelCatalog** | `chat/AIModelCatalog.java` | 获取并缓存可用模型列表 |
| **AssistantProfileChatMemory** | `chat/AssistantProfileChatMemory.java` | 聊天记忆管理（token 上限驱逐） |
| **ChatRequestFlow** | `chat/ChatRequestFlow.java` | 请求生命周期管理 |

#### 支持的 AI Provider

| Provider | API 路由 | 支持模型 |
|----------|---------|---------|

### 5.3 AI 工具集（AIToolSet）

AI 通过 LangChain4j `@Tool` 注解调用这些工具直接操作思维导图：

#### 读取类工具

| 工具 | 功能 |
|------|------|
| `readNodesWithDescendants` | 读取节点及其所有子节点 |
| `fetchNodesForEditing` | 获取可编辑的节点内容 |
| `searchNodes` | 搜索节点（关键字/正则匹配） |
| `getSelectedMapAndNodeIdentifiers` | 获取当前选中的导图和节点 |

#### 写入类工具

| 工具 | 功能 |
|------|------|
| `createNodes` | 创建节点和子树 |
| `edit` | 编辑节点内容（文本/图标/属性/标签/样式/超链接） |
| `deleteNodes` | 删除节点 |
| `moveNodes` | 移动节点相对位置 |
| `createSummary` | 创建摘要节点和括号 |
| `moveNodesIntoSummary` | 将节点移入摘要 |
| `editConnectors` | 编辑节点间连线 |

### 5.4 MCP 服务器（Model Context Protocol）

将导图操作工具通过 **MCP 协议** 暴露为 HTTP 服务，供外部 AI 客户端调用：

- **端口**: 6298
- **协议**: SSE + JSON-RPC 2.0
- **鉴权**: Token 验证（`MCPAuthenticator`）
- **支持客户端**: Claude Desktop、Cursor 等

#### 核心组件

| 组件 | 文件 | 功能 |
|------|------|------|
| **ModelContextProtocolServer** | `mcpserver/ModelContextProtocolServer.java` | HTTP 服务器 |
| **ModelContextProtocolToolRegistry** | `mcpserver/ModelContextProtocolToolRegistry.java` | 工具注册与路由 |
| **ModelContextProtocolToolDispatcher** | `mcpserver/ModelContextProtocolToolDispatcher.java` | 工具调用分发 |

### 5.5 REST API 服务器

将 Freeplane 功能暴露为 RESTful API，供 Web 前端调用：

- **端口**: 6299
- **协议**: HTTP/JSON
- **跨域**: 支持 CORS（`CorsFilter`）
- **线程池**: 4 个固定线程

#### 路由结构

```
/api
├── /map
│   └── GET /current              # 获取当前导图节点树
├── /nodes
│   ├── GET /{nodeId}             # 获取单节点详情
│   ├── POST /create              # 创建子节点
│   ├── POST /edit                # 编辑节点文本
│   ├── POST /delete              # 删除节点
│   └── POST /search              # 搜索节点
└── /ai
    ├── GET /models               # 获取可用 AI 模型列表
    ├── POST /chat                # AI 对话
    ├── POST /expand-node         # 扩展节点
    ├── POST /summarize           # 分支摘要
    ├── POST /tag                 # 自动标签
    ├── POST /smart               # 智能请求（缓冲层）
    └── POST /generate-mindmap    # 一键生成思维导图
```

### 5.6 智能缓冲层（Buffer Layer）

**目标**：用户输入自然语言，系统自动理解并执行最优操作。

#### 核心组件

| 组件 | 文件 | 功能 |
|------|------|------|
| **IBufferLayer** | `buffer/IBufferLayer.java` | 缓冲层标准接口 |
| **BufferLayerRouter** | `buffer/BufferLayerRouter.java` | 识别功能类型，选择缓冲层 |
| **MindMapRequirementAnalyzer** | `buffer/mindmap/MindMapRequirementAnalyzer.java` | 需求理解（生成/展开/摘要） |
| **MindMapPromptOptimizer** | `buffer/mindmap/MindMapPromptOptimizer.java` | 提示词模板库与优化 |
| **MindMapModelRouter** | `buffer/mindmap/MindMapModelRouter.java` | 模型能力画像与智能选择 |
| **MindMapResultOptimizer** | `buffer/mindmap/MindMapResultOptimizer.java` | JSON 验证、质量评估、结果优化 |

#### 处理流程

```
用户输入: "帮我生成一个关于人工智能的思维导图，包含机器学习和深度学习"
  ↓
1. 需求分析 → 类型: MINDMAP_GENERATION, 主题: "人工智能", 语言: "zh"
  ↓
2. 提示词优化 → 选择模板 "mindmap.generation.zh"，填充参数
  ↓
3. 模型选择 → 评分算法选择最优模型（质量/速度/成本）
  ↓
4. 调用 AI → 发送优化后的提示词
  ↓
5. 结果优化 → 验证 JSON、评估质量、清理空节点
  ↓
6. 创建导图 → 调用 MMapController 创建节点树
```

### 5.7 AI 编辑标记

追踪 AI 修改过的节点，在节点旁显示状态图标，支持清除标记操作。

- **位置**: `edits/` 目录
- **功能**: 高亮显示 AI 修改的节点
- **操作**: 清除所有标记

### 6.3 前后端数据流

```
Vue3 前端 (Port 5173)

Vite Proxy (开发环境)
  ↓ 转发
REST API 服务器 (Port 6299)
  ↓ 调用
Freeplane 数据模型 (MapModel/NodeModel)
  ↓ 返回 JSON
前端渲染 (Vue Flow)
```

```typescript
// vite.config.ts
server: {

      secure: false
    }
  }
}
```

```java
map.put("id", node.getID());
```

## 十、部署与运行

### 10.1 完整启动流程

```
1. gradle build
   ↓
2. BIN\freeplane.bat
   ↓ (OSGi 加载 AI 插件)
3. Activator.start()
   ↓
4. 启动 MCP 服务器 → 监听端口 6298
   ↓
5. 启动 REST API 服务器 → 监听端口 6299
   ↓
6. cd freeplane_web && npm run dev
   ↓
7. Vite Dev Server → 监听端口 5173
   ↓
8. 浏览器访问 http://localhost:5173
   ↓
9. 前端通过代理调用后端 API ✅
```

### 10.2 配置文件

#### AI Provider 配置

在 Freeplane 中：
1. 打开 **工具 → 偏好设置 → AI Chat**
2. 配置至少一个 Provider 的 API Key：
   - **OpenRouter**: 输入 API Key（可从 [openrouter.ai](https://openrouter.ai) 获取）
   - **Google Gemini**: 输入 API Key（可从 [Google AI Studio](https://aistudio.google.com) 获取）
   - **Ollama**: 输入服务地址（默认 `http://localhost:11434`）

#### freeplane.properties

位置：`~/.freeplane/1.13.x/freeplane.properties`

**Windows**：`C:\Users\<用户名>\.freeplane\1.13.x\freeplane.properties`

> ⚠️ **注意**: Windows 下需手动创建此文件并确保 UTF-8 编码

### 10.3 端口说明

| 服务 | 端口 | 协议 | 用途 |
|------|------|------|------|
| **MCP Server** | 6298 | SSE + JSON-RPC | 外部 AI 客户端调用 |
| **REST API** | 6299 | HTTP/JSON | Web 前端调用 |
| **Vite Dev** | 5173 | HTTP | 前端开发服务器 |
